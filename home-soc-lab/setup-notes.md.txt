## VirtualBox Troubleshooting Notes

During the Windows 11 VM setup, I ran into repeated black-screen issues after installation.

I troubleshot the issue by:

* Checking VirtualBox storage settings
* Removing the Windows installation ISO after Windows installed
* Confirming the virtual hard disk was still attached
* Adjusting display settings
* Restarting the VM until Windows successfully reached the login screen

This helped me practice basic virtualization troubleshooting, boot troubleshooting, and careful step-by-step problem solving.

## Sysmon Setup

* Tool Installed: Sysmon
* Source: Microsoft Sysinternals
* Installed On: Windows 11 Enterprise Evaluation VM
* Purpose: Collect endpoint telemetry such as process creation events, process termination events, and command-line activity.

### Install Command Used

```powershell
.\Sysmon64.exe -accepteula -i
```

## Sysmon Investigation Evidence

During the lab, I generated command-line activity from PowerShell and reviewed the related Sysmon process creation event.

### Key Event

* Event ID: 1
* Task Category: Process Create
* Image: C:\Windows\System32\net.exe
* Command Line: "C:\WINDOWS\system32\net.exe" user
* Parent Image: C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe
* Parent Command Line: "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe"
* User: DESKTOP-N2D9N5I\socstudent
* Computer: DESKTOP-N2D9N5I
* Screenshot: screenshots/sysmon-event-id-1-net-user-parent-powershell.png

### Analysis Note

The command `net user` lists local user accounts on a Windows system. In this lab, the activity was expected because I manually generated it. In a real SOC environment, this type of command may be reviewed because attackers can use account discovery commands to learn more about a system after gaining access.
